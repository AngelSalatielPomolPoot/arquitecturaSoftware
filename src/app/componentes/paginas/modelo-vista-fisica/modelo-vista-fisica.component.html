Vista
Física

En esta vista se muestra desde la perspectiva de un ingeniero de sistemas todos los componentes físicos del sistema así como las conexiones físicas entre esos componentes que conforman la solución (incluyendo los servicios). Para completar la documentación de esta vista se puede incluir el diagrama de despliegue de UML.

Descripción

La arquitectura física tiene en cuenta principalmente los requisitos no funcionales del sistema, como la disponibilidad, la confiabilidad (tolerancia a fallas), el rendimiento (rendimiento) y la escalabilidad. El software se ejecuta en una red de computadoras o nodos de procesamiento (o simplemente nodos para abreviar). Los diversos elementos identificados (redes, procesos, tareas y objetos) deben mapearse en los distintos nodos.

Esperamos que se utilicen varias configuraciones físicas diferentes: algunas para el desarrollo y las pruebas, otras para la implementación del sistema para varios sitios o para diferentes clientes. Por lo tanto, la asignación del software a los nodos debe ser muy flexible y tener un impacto mínimo en el código fuente en sí.

En última instancia, el software debe residir, instalarse o implementarse en el hardware para su ejecución, y la vista física de la arquitectura del software se centra en cómo el "programa de destino y su tiempo de ejecución dependiente y el software del sistema" se instalan o implementan en las máquinas físicas, y cómo para implementar máquinas y redes Cumplir con los requisitos de confiabilidad y escalabilidad del sistema de software.

También podemos expresar el módulo de destino específico y su estructura de comunicación de manera más clara a través de la vista de arquitectura física de acuerdo con las necesidades de la situación específica
